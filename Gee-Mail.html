Gee-Mail Project:

This is an encrypted messaging system so we do not ever store passwords, message, or passphrases in plaintext message in the databases. The function of this project is mainly two parts. The first part is about register and login. When loaded prompt the user to sign-in or register. When a user registers get their username and password. When a user logs in ask for username and validate password. If login fails, give them corresponding response. Once logged in, tell them the number of messages they have, list the users that have sent them a message. The second part is about sending and receiving the message. After login success, ask if they want to read messages or write messages. When writing message, our system prompts them for the recipient username, the message, and a shared passphrase with that recipient. When reading messages, our system lets them select the message, then prompt them for the shared passphrase with that recipient, then show a plaintext message. This is almost a full stack development which needs to use c+, html, php, css, ajax, sqlite database to develop this application. I gained a lot of interests in this project compared the task that using openvas to scan or build secure system.


Command for makefile:

g++ -o check_login check_login.cpp -lsqlite3 -lgcrypt
./check_login [username] [passwd]

g++ -o register register.cpp -lsqlite3 -lgcrypt
./register [username] [passwd]

g++ -I/usr/local/include/cryptopp -o write_message write_message.cpp -lgcrypt -lsqlite3 -lcryptopp
./write_message [sender] [receiver] ['message'] ['passphrase']

g++ -I/usr/local/include/cryptopp -o read_message read_message.cpp -lgcrypt -lsqlite3 -lcryptopp
./read_message [message_id] ['passphrase']
